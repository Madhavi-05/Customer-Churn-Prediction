# Customer Churn Prediction

## Project Overview

This project focuses on analyzing telecom customer churn and building a machine learning model to predict whether a customer is likely to leave the company.

The project combines:

* Data Cleaning
* Exploratory Data Analysis (EDA)
* Data Visualization
* Feature Engineering
* Machine Learning
* Business Insights

## Problem Statement

Customer churn is one of the biggest challenges for subscription-based businesses. Losing customers reduces revenue and increases acquisition costs.

This project aims to:

* Identify major churn factors
* Analyze customer behavior
* Predict churn using machine learning
* Provide actionable business recommendations

## Dataset Summary

* Total Records: 7,043
* Final Cleaned Records: 7,032
* Features: 21
* Target Variable: Churn

Key features include:

* Customer Demographics
* Subscription Services
* Billing Information
* Contract Details
* Payment Methods

---

## Exploratory Data Analysis

The following analyses were performed:

* Churn Distribution
* Contract vs Churn
* Internet Service vs Churn
* Payment Method vs Churn
* Monthly Charges Analysis
* Tenure Analysis

Key findings:

* Month-to-month customers churn more
* Fiber optic customers show higher churn
* Electronic check users churn more
* High monthly charges increase churn probability

---

## Machine Learning Model

Algorithm Used:

* Logistic Regression

Pipeline:

1. Data Cleaning
2. Missing Value Handling
3. Encoding Categorical Features
4. Train-Test Split
5. Model Training
6. Performance Evaluation

Evaluation Metrics:

* Accuracy Score
* Precision
* Recall
* F1 Score
* Confusion Matrix

Model Accuracy: ~80%

---

## Business Recommendations

* Encourage long-term contracts
* Improve retention of new customers
* Reduce churn among high-billing customers
* Promote automatic payment methods
* Improve service quality for high-risk segments
